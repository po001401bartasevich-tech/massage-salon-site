# Описание страниц сайта массажного салона «MassageRelax»

## 1. Главная (/)
- **Тип доступа:** публичный
- **Основные блоки:**
  1. Шапка: логотип «MassageRelax», меню (Главная, Услуги, Отзывы, Контакты), кнопка «Войти» (если пользователь не авторизован) или аватар с выпадающим меню (если авторизован).
  2. Баннер с приветствием и кнопкой «Записаться онлайн».
  3. Секция «Популярные услуги»: 4 карточки услуг (название, цена, длительность, изображение). Данные загружаются с API: GET /api/services?popular=true.
  4. Секция «Последние отзывы»: 3 отзыва с именем, рейтингом и текстом. Данные: GET /api/reviews?limit=3.
  5. Футер: адрес, телефон, email, ссылки на социальные сети.
- **Динамический контент:** блоки 3 и 4 (загружаются с сервера).

## 2. Аутентификация (/auth/*)
- **Тип доступа:** публичный
- **Страницы:**
  - `/auth/login` – форма входа: поля «Email» и «Пароль», кнопка «Войти», ссылки на регистрацию и восстановление пароля.
  - `/auth/register` – форма регистрации: поля «Имя», «Email», «Пароль», «Телефон», кнопка «Зарегистрироваться».
  - `/auth/forgot-password` – форма запроса восстановления: поле «Email», кнопка «Отправить».
  - `/auth/reset-password` – форма нового пароля: поле «Новый пароль», «Подтверждение пароля», кнопка «Сохранить» (доступна по ссылке из письма).
- **Динамический контент:** отсутствует, все формы статичны, данные отправляются на API.

## 3. Каталог услуг (/services)
- **Тип доступа:** публичный
- **Основные блоки:**
  1. Шапка (аналогично главной).
  2. Заголовок «Наши услуги».
  3. Блок фильтров: по категории (например, «Релакс», «Лечебный», «Спортивный») и по цене (ползунок или диапазон). Фильтрация реализуется через запросы к API.
  4. Список услуг в виде карточек (название, цена, длительность, изображение). Данные: GET /api/services (с возможными параметрами фильтрации).
- **Динамический контент:** список услуг, результаты фильтрации.

## 4. Детальная страница услуги (/services/:id)
- **Тип доступа:** публичный
- **Основные блоки:**
  1. Шапка.
  2. Крупное изображение услуги.
  3. Название, подробное описание, цена, длительность.
  4. Кнопка «Записаться» (если пользователь авторизован, ведёт на страницу бронирования; если нет – предлагает войти или зарегистрироваться).
  5. Блок «Отзывы на эту услугу»: список отзывов с оценками и текстом. Данные: GET /api/reviews?serviceId=:id.
- **Динамический контент:** данные услуги, отзывы.

## 5. Личный кабинет (/profile)
- **Тип доступа:** приватный (только для авторизованных)
- **Основные блоки:**
  1. Шапка с именем пользователя.
  2. Навигация по личному кабинету: вкладки «Мои данные» и «Мои записи».
  3. Вкладка «Мои записи» по умолчанию: список будущих и прошедших записей (карточки или таблица). Для каждой записи: дата, время, услуга, мастер, статус (подтверждена, отменена, выполнена), кнопка отмены (если запись ещё актуальна). Данные: GET /api/bookings/my.
  4. Вкладка «Мои данные»: форма с именем, телефоном, email (возможно редактирование), кнопка «Сохранить изменения». 
- **Динамический контент:** список записей, данные пользователя.

## 6. Онлайн-запись (/booking)
- **Тип доступа:** приватный
- **Основные блоки:**
  1. Шапка.
  2. Пошаговая форма (можно реализовать на одной странице или с переходами):
     - Шаг 1: выбор услуги из выпадающего списка (данные GET /api/services).
     - Шаг 2: выбор мастера (если есть) и даты. При выборе даты подгружаются доступные временные слоты через API (например, GET /api/slots?serviceId=...&date=...).
     - Шаг 3: подтверждение выбранных данных и кнопка «Подтвердить запись» (POST /api/bookings).
- **Динамический контент:** список услуг, доступные слоты времени.

## 7. Отзывы (/reviews)
- **Тип доступа:** публичный (добавление отзыва – приватное)
- **Основные блоки:**
  1. Шапка.
  2. Заголовок «Отзывы наших клиентов».
  3. Список отзывов с пагинацией. Каждый отзыв содержит: имя пользователя, дату, оценку (звёзды), текст отзыва, возможно название услуги (если отзыв на конкретную услугу). Данные: GET /api/reviews.
  4. Если пользователь авторизован – форма добавления отзыва: выбор услуги (из списка), оценка (1–5), текстовое поле, кнопка «Оставить отзыв».
- **Динамический контент:** список отзывов, форма (показывается только авторизованным).

## 8. Контакты (/contacts)
- **Тип доступа:** публичный
- **Основные блоки:**
  1. Шапка.
  2. Текст с адресом салона, телефоном, email, режимом работы.
  3. Карта (можно встроить Яндекс.Карты или Google Maps статическим изображением).
  4. Форма обратной связи: поля «Имя», «Email», «Сообщение», кнопка «Отправить». Отправка данных через POST /api/feedback (сообщение уходит администратору на email).
- **Динамический контент:** форма обратной связи (отправка данных).

## 9. Поиск (/search)
- **Тип доступа:** публичный
- **Основные блоки:**
  1. Шапка.
  2. Строка поиска: поле ввода, кнопка «Найти».
  3. Результаты поиска: выводятся в виде карточек услуг (как в каталоге). Данные: GET /api/search?q=запрос.
  4. Если результатов нет – сообщение «Ничего не найдено» и предложение посмотреть каталог.
- **Динамический контент:** результаты поиска.

## 10. Админ-панель (/admin)
- **Тип доступа:** приватный (только для пользователей с ролью admin)
- **Основные блоки:**
  1. Шапка с навигацией по разделам админки.
  2. Разделы (доступны через меню или вкладки):
     - **Управление услугами:** таблица услуг с возможностью добавить, редактировать, удалить (CRUD). Данные: GET /api/admin/services, POST, PUT, DELETE.
     - **Управление мастерами:** аналогично.
     - **Управление записями:** просмотр всех записей, изменение статуса.
     - **Управление отзывами:** просмотр, публикация/скрытие, удаление.
- **Динамический контент:** все данные загружаются через API с правами администратора.
